iOS SDK Documentation

SMUser

Superclass:
NSObject
Declared In:
SessionM.h

Introduction

Provides information about user such as unclaimed achievement count, point balance, opt out status, etc.

Discussion

When user data is changed as a result of user earning a new achievement or opting out of the program SessionMDelegate method sessionM:didUpdateUser: is called to notify the application about this event.


Properties

achievements

Returns an array of SMAchievementData objects.

achievementsList

Returns an array of SMAchievementData objects representing the user's achievement forecast.

claimableAchievements

Returns the subarray of achievementsList that contains all achievements that the user can currently claim.

isLoggedIn

Returns user's logged in status. This value is set to true for a registered user who has also verified their password for the current session through the portal - or when either signUpUserWithData: or logInUserWithEmail:password: is called.

isOptedOut

Returns or sets user opt out status.

isRegistered

Returns user's registered status. This value is set to true when the account corresponds to a registered user. The user can register for an account through the portal - or when either enrollWithData:, enrollWithEmail:YOB:, signUpUserWithData: or logInUserWithEmail:password: is called.

pointBalance

Returns user's current points balance.

unclaimedAchievementCount

Returns user's unclaimed achievement count.

unclaimedAchievementValue

Returns user's unclaimed achievement value.

Discussion

Use this property to programmatically opt-in or opt-out user or query for this status. When opted out tha application can still log actions but they would not trigger an achievement display. User portal can still be presented in opted out state and users can opt in into the propgram from it.
