Introduction

View controller for portal load screen.

Discussion

This class encapsulates the view that is displayed when portal content is being loaded, along with the state and behavior associated with that view. A developer who is implementing a custom load screen must subclass this class and override the updateViewForLoadStatus: method to use their custom view.

reloadPortalContent

Attempts to reload the portal content after a failed attempt. 

- (void)reloadPortalContent;
Discussion

Called by the developer when implementing a custom load screen. Note: this method has no effect unless the current load status is SMLoaderControllerLoadStatusFailed.

updateViewForLoadStatus:

Updates the view based on the current load status. 

- (void)updateViewForLoadStatus:(SMLoaderControllerLoadStatus)status;
Parameters
status

Current load status.

Discussion

Called when the value of loadStatus changes. This method should not be called directly, but the developer should override it to update their custom view based on the current load status.
